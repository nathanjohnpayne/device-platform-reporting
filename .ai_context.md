## Standard Compliance Notes

- **No `.cursor/` or `.vscode/` directories:** Only `.claude/` exists. No Cursor or VS Code config in this repo.
- **`dist/` present:** webpack build artifacts. Never edit directly. The `check_dist_not_modified` CI script enforces this.
- **`tests/` placeholder:** The Jest test suite is configured via `jest.config.js` at the root, with test files co-located. The root `tests/` directory is a standard placeholder per the AI Agent Tooling Standard.
- **`scripts/ci/` present but CI pipeline not connected:** CI scripts exist per the standard. `scripts/check-no-public-secrets.mjs` runs as part of `npm test`.
- **`.claude/worktrees/`:** Claude Code's isolated git worktree directory (machine-generated). The CI check for tool folder instructions excludes this directory from scan.
- **No Cloud Functions:** This is a pure client-side app. The product spec mentions Cloud Functions, but they have not been implemented. See AGENTS.md Section 2 for the known gap.

## Related Repositories

- **`ai_agent_repo_template`** (sibling directory) — Canonical template per the AI Agent Tooling Standard. Contains the `gcloud` wrapper, `op-firebase-deploy` / `op-firebase-setup` scripts, the 1Password-backed impersonation deployment guide, and the full standard definition. Reference when creating new projects or when deployment questions extend beyond this repo.
- **Deploy-auth invariant** — This repo intentionally uses the shared 1Password-backed `Private/GCP ADC` source credential for routine human `gcloud` and deploy work. Do not migrate it back to ADC-first or deploy-key auth without explicit human approval.
